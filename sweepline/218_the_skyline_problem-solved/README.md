# 218 · The Skyline Problem

给定建筑物列表 `buildings[i] = [left, right, height]`，返回它们组成的天际线**关键拐点**列表 `[[x, y], ...]`。每个拐点是一段水平线段的左端点；末尾点 `y = 0` 标记结束。输出中不能有连续等高的拐点。

## Example
- `[[2,9,10],[3,7,15],[5,12,12],[15,20,10],[19,24,8]]`
  → `[[2,10],[3,15],[7,12],[12,0],[15,10],[20,8],[24,0]]`
- `[[0,2,3],[2,5,3]]` → `[[0,3],[5,0]]`

## Constraints
- `1 <= buildings.length <= 10^4`
- `0 <= left < right <= 2^31 - 1`
- `1 <= height <= 2^31 - 1`
- `buildings` 按 `left` 升序

## 思路（扫描线 + 最大堆）

**核心**: 把每个建筑拆成两个事件 —— "进入"和"离开"。从左到右扫描事件点，维护"当前活跃高度"的最大堆，**当堆顶高度变化时**，就产生一个拐点。

**步骤**:
1. **构造事件**: 对每个 `[l, r, h]` 产出两个事件 `(l, -h, r)`（进入）和 `(r, 0, 0)`（离开标记）
2. **排序事件**: 按 `x` 升序；同 `x` 时让"进入"在前（`-h` 越小越前 → 高的先进入），避免在同一 x 处先输出低再输出高
3. **扫描**:
   - 进入事件 → 把 `(h, r)` 推入最大堆
   - 离开事件 → **懒删除**：堆顶若 `r <= 当前 x`，弹掉（已过期）
   - 当前最大高度 = 堆顶 `h`（若堆空则 0）
   - 若与上一个高度不同 → 加入答案 `[x, h]`，更新上一个高度
4. 时间 `O(n log n)`，空间 `O(n)`

**关键点**:
- **为什么用懒删除**: 标准堆不支持 O(log n) 删除任意元素，所以让过期项"自然浮上来再丢"
- **为什么进入用 `-h`**: 让排序时同一 x 的"高建筑先进入"，避免输出多余拐点
- **去重**: 用 `prevHeight` 记录上次输出高度，相同则跳过

## 易错点
- 多个建筑同 x 进入：必须**先处理所有进入，再判断高度变化**（否则会输出中间态）
- 一个进入一个离开同 x：进入先 → 先把新高度入堆 → 再让旧高度懒删除 → 避免假性下降
- 末尾点 `y=0`：所有建筑都离开后，堆空，高度 0 ≠ prev → 自然输出

## 替代解法（仅了解）
- **分治** O(n log n): 类似归并，左右各算 skyline 再合并 —— 思路漂亮但代码长
- **TreeMap (Java)** / **SortedList (Python)**: 用有序结构维护活跃高度计数，可 O(log n) 真删除，避免懒删除复杂度退化

> 面试首选 **扫描线 + 最大堆 + 懒删除**，代码最短且模式可复用到 253、2402。
