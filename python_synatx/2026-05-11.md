# Python / JS Syntax — 2026-05-11

## JS: `Array.prototype.push`

### 基本用法

```js
const arr = [1, 2]
arr.push(3)           // arr = [1,2,3]，返回新长度 3
arr.push(4, 5)        // 一次加多个：arr = [1,2,3,4,5]
arr.push(...[6, 7])   // 展开数组：arr = [1,2,3,4,5,6,7]
arr.push()            // 0 个参数也合法，什么都不做
```

### 参数

- 数量：任意（0 个、多个都可）
- 类型：任何能放进数组的值（number / string / object / array / function ...）
- TypeScript 中类型必须匹配数组的元素类型

### 返回值

**新长度（number），不是数组本身**。所以：

```js
arr = arr.push(x)   // ❌ arr 会变成数字
```

### 加数组：要不要展开？

```js
const res = []
const interval = [1, 2]

res.push(interval)      // res = [[1, 2]]   整个数组作为一个元素
res.push(...interval)   // res = [1, 2]     展开成多个元素
```

- `res` 是 `number[][]` → `res.push(interval)` ✅（如 LeetCode 057 Insert Interval）
- `res` 是 `number[]`，要拼接 → `res.push(...other)`

## JS: `push` vs `concat`

| | `push` | `concat` |
|---|---|---|
| 修改原数组 | ✅ | ❌ |
| 返回值 | 新长度 | 新数组 |
| 加数组时 | 整个数组作为一个元素（除非展开） | 自动拆开拼接 |

```js
const a = [1, 2]
const b = [3, 4]

a.push(b)        // a = [1, 2, [3, 4]]
a.push(...b)     // a = [1, 2, 3, 4]

const c = a.concat(b)  // c = [1, 2, 3, 4]，a 不变
```

## Python 对应：`append` / `extend` / `+`

| JS | Python | 修改原数组 | 返回值 |
|---|---|---|---|
| `arr.push(x)` | `arr.append(x)` | ✅ | `None` |
| `arr.push(...other)` | `arr.extend(other)` 或 `arr += other` | ✅ | `None` / `arr` |
| `a.concat(b)` | `a + b` | ❌ | 新列表 |

```python
a = [1, 2]
b = [3, 4]

a.append(b)   # a = [1, 2, [3, 4]]   整个塞进去
a.extend(b)   # a = [1, 2, 3, 4]     展开
a += b        # 同 extend
c = a + b     # 新列表，a 不变
```

### 口诀

- `push` ↔ `append`（加单个元素）
- `push(...x)` ↔ `extend(x)`（展开）
- `concat` ↔ `+`（不改原数组）

### 坑

- `a.extend(b)` 返回 `None`，别 `a = a.extend(b)`，否则 `a` 变 `None`
- 同理 JS 的 `a = a.push(x)` 会让 `a` 变成数字

## LeetCode 057 Insert Interval: 标志位别忘了置 true

```ts
function insert(intervals: number[][], newInterval: number[]): number[][] {
    const res = []
    let inserted = false
    for (const interval of intervals) {
        if (interval[1] < newInterval[0]) {
            res.push(interval)
        } else if (interval[0] > newInterval[1]) {
            if (!inserted) {
                res.push(newInterval)
                inserted = true        // ⚠️ 必须置 true，否则后续每次都会重复 push
            }
            res.push(interval)
        } else {
            newInterval[0] = Math.min(newInterval[0], interval[0])
            newInterval[1] = Math.max(newInterval[1], interval[1])
        }
    }
    if (!inserted) res.push(newInterval)
    return res
}
```

**反例**（如果忘了 `inserted = true`）：
`intervals=[[1,2],[6,7],[8,10]], newInterval=[3,4]` → 结果会变成 `[[1,2],[3,4],[6,7],[3,4],[8,10]]`，`newInterval` 被 push 了两次。

## JS: `flatMap` —— 一个元素展开成多个

`map` 是 1→1，`flatMap` 是 1→N（自动拍平一层）。扫描线把每个 interval 拆成两个事件点时最自然：

```ts
const points = airplanes.flatMap(air => [
    { index: air[0], value: 1 },
    { index: air[1], value: -1 },
])
```

对比：

- `map` → 得到 `{...}[][]`，还要 `.flat()`
- `flatMap` → 直接得到 `{...}[]`

## Python: 嵌套列表推导式 —— 怎么断句

```python
points = [p for air in airplanes for p in (start_pt, end_pt)]
```

**口诀：从左到右按 for 子句的顺序读，跟普通 for 完全一一对应**。

```
[  p             for air in airplanes    for p in (...)  ]
   ↑             ↑                       ↑
  收集的东西    外层循环                内层循环
```

等价展开：

```python
points = []
for air in airplanes:           # 外层
    for p in (start_pt, end_pt): # 内层
        points.append(p)         # 最左表达式
```

⚠️ 不是从右到左，**Python 推导式按你写的顺序从左往右嵌套**。

第二个 `for` 后面只要是**可迭代对象**就行（tuple / list / generator 都可），手工临时构造一个 tuple 是最常用的展开技巧。

## Python: `sort` / `sorted`

| 写法 | 改原数组 | 返回值 |
|---|---|---|
| `lst.sort()` | ✅ | `None` |
| `sorted(iterable)` | ❌ | 新 list |

```python
lst.sort()                           # 升序
lst.sort(reverse=True)               # 降序
lst.sort(key=lambda x: x['index'])   # 按 key 排
new = sorted(lst, key=...)           # 不改原数据
```

⚠️ `lst.sort()` 返回 `None`，**不能** `b = lst.sort()`，要新列表用 `sorted()`。

### Python 没有 comparator

JS 有 `(a, b) => ...` 比较函数，**Python 3 移除了 `cmp` 参数**，多字段排序要用 **key 返回 tuple**：

```ts
// JS：告诉语言怎么比
arr.sort((a, b) => a.index - b.index || a.value - b.value)
```

```python
# Python：把要比的字段按优先级打包成 tuple
lst.sort(key=lambda x: (x['index'], x['value']))
```

**tuple 比较是逐元素的**，从左到右先决出胜负就停：

```python
(1, 5) < (2, 0)   # True  → 第一位定胜负
(1, 5) < (1, 9)   # True  → 第一位平，看第二位
```

正好 = "先按字段 A，A 相同再按字段 B"。

---

## `key=lambda` 是什么意思

- `key=` —— sort 的参数，意思"用我提取的值去比"，传入的是**一个函数**
- `lambda x: x['index']` —— 匿名函数，等价于：

  ```python
  def get_index(x): return x['index']
  ```

整个 `lambda x: x['index']` 是**不可分割的整体**，作为函数对象传给 `key=`：

```python
points.sort(key = lambda x: x['index'])
#           └┬┘   └────────┬────────┘
#         参数名    一个函数对象
```

`lambda` 的语法边界：从 `lambda` 到遇到 `,` / `)` / 行尾结束。

### JS vs Python 排序传函数的根本差异

- JS: `sort((a,b) => ...)` 传**比较函数**（看两个元素）
- Python: `sort(key=fn)` 传**提取函数**（只看一个元素，提取排序依据）

## Python: dict 用 `[]`，object 用 `.` —— 不能混

```python
# dict
d = {'value': 1}
d['value']   # ✅
d.value      # ❌ AttributeError

# object（class 实例）
class Point:
    def __init__(self, v): self.value = v
p = Point(1)
p.value      # ✅
p['value']   # ❌
```

题目里 `Interval` 是 class，所以是 `air.start` / `air.end`，**不能** `air[0]`。

### Python 造"轻量对象"的几种方式

```python
# 1. dataclass（最现代）
from dataclasses import dataclass
@dataclass
class Point:
    index: int
    value: int

# 2. NamedTuple（不可变 + 可解包）
from typing import NamedTuple
class Point(NamedTuple):
    index: int
    value: int

# 3. 普通 class
class Point:
    def __init__(self, i, v):
        self.index, self.value = i, v
```

### 选什么容器

| 选 | 场景 |
|---|---|
| `tuple` `(1, -1)` | 字段少、固定顺序，最轻 |
| `dict` | 字段名重要、动态 |
| `NamedTuple` / `dataclass` | 固定字段 + 想 `.field` 访问 |
| `class` | 需要方法/复杂行为 |

## JS `{}` vs Python `{}` —— 根本不是一种东西

| 概念 | JS | Python |
|---|---|---|
| 键值映射（字典用） | `Map`（推荐）/ `{}` | `dict` `{}` |
| 取值 | `o.a` 或 `o['a']` 都行 | 只能 `d['a']` |
| 类的实例 | `class` 实例 | `class` 实例 |
| 取属性 | `instance.x` | `instance.x` |

### 关键区别

- **JS 的 `{}` 是 Object 实例** —— 既是数据容器又是对象，`.` 和 `[]` 都行（一物多用）
- **Python 的 `{}` 是 dict 实例** —— 纯映射，只能 `[]`，对象要用 class（专物专用）

```js
// JS：{} 真的是 Object
const o = {}
o instanceof Object  // true
o.toString()         // '[object Object]'  ← 继承自 Object.prototype
```

```python
# Python：{} 是 dict，不是泛用 object
d = {}
type(d)  # <class 'dict'>
d.name   # ❌ AttributeError
```

### 为什么 JS 的 `{}` 当字典用有坑（社区推荐 `Map`）

| | `{}` plain object | `Map` |
|---|---|---|
| key 类型 | 只能 string/symbol | 任何类型 |
| 原型链污染 | 有 `toString` 等继承属性 | 干净 |
| 遍历顺序 | 数字 key 会被特殊排序 | 严格按插入顺序 |
| 大小 | `Object.keys(o).length` | `.size` |

### 一图记牢

```
              动态映射         结构化对象
            ┌─────────────┬─────────────┐
   Python   │    dict     │    class    │   ← 严格分开
            ├─────────────┼─────────────┤
     JS     │  Map / {}   │    class    │   ← {} 模糊地带
            └─────────────┴─────────────┘
```

## LeetCode 391 Number of Airplanes — 修正版回顾

踩到的坑：

1. `air[0]` ❌ → `air.start` ✅（Interval 是对象不是 tuple）
2. `maxPlane, sum = 0` ❌ → `maxPlane, total = 0, 0` ✅（不能把 int 解包给两个变量）
3. `point.values` ❌ → `point['value']` ✅（dict 用 `[]`，且字段是单数 `value`）
4. `sum` 别做变量名 —— 遮蔽内置函数

最 Pythonic 写法（用 tuple 替代 dict，连 `key=` 都省了）：

```python
def count_of_airplanes(self, airplanes: List[Interval]) -> int:
    points = [pt for air in airplanes
                 for pt in ((air.start, 1), (air.end, -1))]
    points.sort()  # tuple 默认按 (index, value) 多字段排序
    maxPlane = total = 0
    for _, v in points:
        total += v
        maxPlane = max(maxPlane, total)
    return maxPlane
```

## Python: `heapq` —— 标准库优先队列（学到 PQ 时再看）

### 不是类，是「操作 list 的一组函数」

```python
import heapq

pq = []                       # 堆就是一个普通 list
heapq.heappush(pq, 5)         # 入堆，原地改 list
heapq.heappush(pq, 1)
pq[0]                         # 看堆顶（不弹），1
heapq.heappop(pq)             # 弹出最小，返回 1
len(pq)                       # 当 list 用就行
```

跟 JS / Java 不一样，Python **没有** `PriorityQueue` 类。`heapq` 提供函数，list 充当容器。

### 常用 API

| API | 作用 | 复杂度 |
|---|---|---|
| `heapq.heappush(h, x)` | 入堆 | O(log n) |
| `heapq.heappop(h)` | 弹出最小，返回值 | O(log n) |
| `h[0]` | 看堆顶（不弹） | O(1) |
| `heapq.heapify(lst)` | 原地变堆 | O(n) |
| `heapq.heappushpop(h, x)` | 先 push 再 pop（更快） | O(log n) |
| `heapq.heapreplace(h, x)` | 先 pop 再 push（更快） | O(log n) |
| `heapq.nlargest(k, lst)` | 最大的 k 个 | O(n log k) |
| `heapq.nsmallest(k, lst)` | 最小的 k 个 | O(n log k) |

### 命名坑

`heap` + `push` 拼一起 → **`heappush`** / **`heappop`**（中间两个 `p`）。

容易写错：
- ❌ `heapq.heapop(pq)`  → ✅ `heapq.heappop(pq)`
- ❌ `heap_push`         → ✅ `heappush`

### 没有最大堆？取负

```python
heapq.heappush(pq, -x)
top = -heapq.heappop(pq)
```

### 存复杂对象：用 tuple

按第一个字段比，相同时比第二个：

```python
heapq.heappush(pq, (end_time, meeting_id))
```

若对象不可比，加个 counter 防崩：

```python
import itertools
counter = itertools.count()
heapq.heappush(pq, (priority, next(counter), obj))
```

### `nlargest` / `nsmallest` 面试能用吗

**可以**。内部就是 size-k 最小堆 / 最大堆，O(n log k)，等价于手写 Top K。

用之前建议说一句"我知道这是 O(n log k) 的 size-k 堆"，证明你懂原理就行。

题目要求"实现 Top K / 不准用库"时再手写。

### 三种 Top K 对比

```python
sorted(nums, reverse=True)[:k]   # O(n log n)
heapq.nlargest(k, nums)          # O(n log k) ← 推荐
# 手写 size-k 最小堆
h = []
for x in nums:
    heapq.heappush(h, x)
    if len(h) > k: heapq.heappop(h)
```

### Meeting Rooms II 模板

```python
import heapq
class Solution:
    def minMeetingRooms(self, intervals: List[List[int]]) -> int:
        intervals.sort(key=lambda x: x[0])
        pq = []  # 存「正在进行的会议」的结束时间
        for start, end in intervals:
            if pq and pq[0] <= start:
                heapq.heappop(pq)    # 最早结束的房间空了 → 复用
            heapq.heappush(pq, end)  # 当前会议占一间房
        return len(pq)
```

**关键直觉**：堆里每个元素 = 一间正在被占用的房间。
- 能复用 → `pop + push`，大小不变
- 不能复用 → 只 `push`，大小 +1
- **永远不减少** → 最终 `len(pq)` = 历史峰值 = 答案
