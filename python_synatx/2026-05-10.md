## JS: `Array(n)` vs `Array.from`

核心区别：`Array(n)` 创建的是**稀疏数组**（empty slots），`Array.from` 创建的是**密集数组**。

```ts
const a = new Array(3);                     // [ <3 empty items> ]
const b = Array.from({length: 3});          // [undefined, undefined, undefined]
const c = Array.from({length: 3}, () => 0); // [0, 0, 0]
```

### 行为差异

```ts
// map / forEach 会跳过 empty slot
new Array(3).map((_, i) => i);        // [ <3 empty items> ]  ❌ 没生效
Array.from({length: 3}, (_, i) => i); // [0, 1, 2]            ✅
```

但 `.fill()` **不跳过** empty，所以这两个等价：

```ts
new Array(3).fill(0);                 // [0, 0, 0]
Array.from({length: 3}, () => 0);     // [0, 0, 0]
```

### 二维矩阵的关键陷阱

```ts
// ❌ 三行共享同一个内部数组（fill 把同一个引用复制了 3 次）
new Array(3).fill(new Array(3).fill(0));

// ✅ 每行独立创建
Array.from({length: 3}, () => new Array(3).fill(0));
```

### 记忆口诀

- 一维填充 → `Array(n).fill(x)` 简洁够用
- 一维带索引 → `Array.from({length: n}, (_, i) => ...)`
- 二维矩阵 → 必须用 `Array.from`，避免引用共享

## JS: `{length: 3}` 是什么 —— 类数组对象

`{length: 3}` 是一个**类数组对象**（array-like object）—— 普通对象，但因为有 `length` 属性，可以被当成数组来遍历。

`Array.from` 的工作原理：接受任何有 `length` 属性的东西，按 `0, 1, 2, ..., length-1` 的顺序去取值，取不到就给 `undefined`：

```ts
Array.from({length: 3});
// 取 obj[0], obj[1], obj[2] → 都是 undefined
// 结果：[undefined, undefined, undefined]

Array.from({length: 3, 0: 'a', 1: 'b'});
// 取到 obj[0]='a', obj[1]='b', obj[2]=undefined
// 结果：['a', 'b', undefined]
```

### 其他类数组对象的例子

```ts
Array.from('abc');                            // ['a','b','c']  字符串有 length
Array.from(document.querySelectorAll('div')); // NodeList 有 length
function f() { return Array.from(arguments); } // arguments 有 length
```

### 含义

`Array.from({length: n}, fn)` = **"我要一个长度为 n 的数组，每个元素用 fn 生成"**。`{length: n}` 只是告诉它"长度多少"，本身没有真实数据。

等价的另一种写法：

```ts
Array.from({length: 3}, (_, i) => i);
// 等价于
Array(3).fill(0).map((_, i) => i);
// 必须先 fill 把 empty 填掉，map 才会执行
```

## Python: 列表初始化 `[0] * n` vs `[0 * n]`

```python
[0] * 5     # → [0, 0, 0, 0, 0]   长度为 5 的零数组
[0 * 5]     # → [0]               先算 0*5=0，再包成单元素列表
```

初始化长度为 `n` 的零数组必须用 `[0] * n`，方括号要包住 `0`，`*` 在外面。

### 可变元素的陷阱

```python
[0] * 3        # ✅ int 不可变，安全
[[]] * 3       # ❌ 三个元素是同一个 list 的引用
               #    修改任一个会影响全部
[[] for _ in range(3)]   # ✅ 每个独立的空 list
```

口诀：**不可变元素用 `* n`，可变元素用列表推导式**。

## Python: 迭代解包 (iterable unpacking)

### 两种常见写法对比

```python
# 1. 直接对元素解包：元素本身就是多字段结构
for start, end, value in updates:      # updates = [[1,3,2], [2,4,3], ...]
    ...

# 2. enumerate 解包：人工造出 (索引, 元素) 对
for index, num in enumerate(res):
    ...
```

| 写法 | 解包来源 | 用途 |
|---|---|---|
| `for a, b, c in items` | 元素本身是 3 元组/列表 | 拆字段 |
| `for i, x in enumerate(arr)` | `enumerate` 造出 `(i, x)` | 既要索引又要元素 |

本质都是**同一个语法**：右边能拆成左边对应数量的变量。

### 数量必须匹配，否则报错

```python
a, b = [1, 2, 3]
# ValueError: too many values to unpack (expected 2)

a, b, c = [1, 2]
# ValueError: not enough values to unpack (expected 3, got 2)
```

### 变长解包：用 `*` 收集多余

```python
a, *rest = [1, 2, 3, 4]            # a=1, rest=[2,3,4]
first, *middle, last = [1,2,3,4,5] # first=1, middle=[2,3,4], last=5
a, b, *_ = [1, 2, 3, 4]            # _ 表示不关心，丢弃
```

### 嵌套解包

```python
for (start, end), value in [((1,3), 2), ((2,4), 3)]:
    # start=1, end=3, value=2
```

### 元素长度不一致就别用固定解包

```python
data = [[1,2], [1,2,3]]
for a, b in data:    # 第二轮报错
    ...
# 改成：for item in data，循环体内判断
```

## Python: `sum` 作为变量名会遮蔽内置函数

```python
sum = 0                    # ❌ 遮蔽了内置 sum()
for num in res:
    sum += num
# 之后再想用 sum([1,2,3]) 就报错

# ✅ 推荐改名
cumulative = 0
total = 0
s = 0
```

同类要避免的内置名：`list`, `dict`, `set`, `str`, `int`, `id`, `type`, `input`, `max`, `min`, `filter`, `map`。

## LeetCode 370 Range Addition: 差分数组 + 前缀和

```python
def getModifiedArray(self, length: int, updates: List[List[int]]) -> List[int]:
    res = [0] * length
    for start, end, value in updates:
        res[start] += value
        stop = end + 1
        if stop < length:
            res[stop] -= value

    cumulative = 0
    for index, num in enumerate(res):
        cumulative += num
        res[index] = cumulative

    return res
```

**核心思路**：
- 差分阶段：区间 `[start, end]` 加 `value` → 只在 `start` 处 `+value`，在 `end+1` 处 `-value`
- 前缀和阶段：对差分数组求前缀和，还原成最终结果
- 时间复杂度从 `O(updates × length)` 降为 `O(updates + length)`
