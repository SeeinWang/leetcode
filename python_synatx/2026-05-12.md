## Python `heapq` 模块

### 核心特点

Python 的 `heapq` **不是一个堆类**，而是**一组操作普通 list 的函数**。`heapq` 维护的不变量：list 满足堆序（父 ≤ 子）。

```python
import heapq

heap = []                  # 就是普通 list！
heapq.heappush(heap, 5)
print(type(heap))          # <class 'list'>
print(heap[0])             # 直接索引访问堆顶 O(1)
```

**Python 只提供 min-heap（小根堆）**。`heap[0]` 永远是当前最小元素。

### 完整 API 速查

| 想做的事 | 写法 | 复杂度 |
|---|---|---|
| 创建空堆 | `heap = []` | O(1) |
| 把已有 list 变成堆 | `heapq.heapify(lst)` | O(n) |
| push | `heapq.heappush(heap, x)` | O(log n) |
| pop 最小 | `heapq.heappop(heap)` | O(log n) |
| **peek 最小** | **`heap[0]`** | **O(1)** |
| push + pop（一步） | `heapq.heappushpop(heap, x)` | O(log n) |
| pop + push（一步） | `heapq.heapreplace(heap, x)` | O(log n) |
| 取最小 N 个 | `heapq.nsmallest(n, lst)` | O(n log k) |
| 取最大 N 个 | `heapq.nlargest(n, lst)` | O(n log k) |
| 大小 | `len(heap)` | O(1) |
| 判空 | `if heap:` 或 `if not heap:` | O(1) |

### 为什么没有 peek()？

因为 heap 就是 list，`heap[0]` 在物理上就是堆顶位置。Python 的设计哲学是不包一层 class，直接给函数。

### 安全 vs 危险的操作

```python
# ✅ 读取自由
heap[0]            # peek
len(heap)          # 大小
if heap: ...       # 判空
for x in heap: ... # 遍历（顺序不是有序的！）

# ❌ 这些会破坏堆序
heap.append(99)    # 不维护堆序
heap.pop()         # 弹出末尾，不是堆顶
heap[2] = 100      # 直接改值
heap.sort()        # 技术上可以但没意义
```

**只能用 `heapq.*` 函数修改堆**。

## 用 min-heap 模拟 max-heap

Python 没有 max-heap，标准技巧：**存负数**。

```python
heap = []
for x in [10, 15, 8]:
    heapq.heappush(heap, -x)   # 实际推入 [-10, -15, -8]

# heap[0] = -15  (min-heap 找最小 → -15)
# 还原: -heap[0] = 15  ← 原数据中的最大值
```

逻辑：
- 原数据中的"最大" → 取负后变成"最小"
- min-heap 把它放堆顶
- 取出来再取负还原

### 元组比较的坑

```python
heappush(heap, (-15, 9))
heappush(heap, (-10, 7))
heappush(heap, (-15, 3))

# heap[0] = (-15, 3)
# Python 元组按字典序逐位比：先比 -15 == -15，再比 3 < 9
```

如果第二位会影响排序但你不在乎它，要么加序号防冲突，要么用 `dataclass(order=False)`。

## `heappushpop` vs `heapreplace`

两个"一步完成 push + pop"的优化函数。

| 函数 | 顺序 | 返回值 | 新值 x 可能立刻被弹出？ |
|---|---|---|---|
| `heappushpop(heap, x)` | 先 push 再 pop | 操作后的最小值 | ✅ 可能（x 比堆顶小时） |
| `heapreplace(heap, x)` | 先 pop 再 push | 旧的最小值 | ❌ 一定留下 |

### `heappushpop` 例子

```python
heap = [3, 5, 7]
heapq.heappushpop(heap, 1)
# 先 push 1 → [1,3,5,7]，再 pop → 弹出 1
# 返回 1，堆变回 [3,5,7]   ← 没变化！

heapq.heappushpop(heap, 10)
# 先 push 10 → [3,5,7,10]，再 pop → 弹出 3
# 返回 3，堆变成 [5,7,10]
```

### `heapreplace` 例子

```python
heap = [3, 5, 7]
heapq.heapreplace(heap, 1)
# 先 pop → 弹出 3，再 push 1
# 返回 3，堆变成 [1,5,7]   ← 1 留下来了

# ⚠️ 空堆调用会报错（先 pop 空堆）
```

### 为什么需要这两个

性能：分开调用走两次堆调整，合并版只走一次。

```python
# 慢：两次堆调整
heapq.heappush(heap, x)
result = heapq.heappop(heap)

# 快：一次调整
result = heapq.heappushpop(heap, x)
```

### 典型应用：维护 Top-K

求前 K 大 → 维护**大小为 K 的 min-heap**：

```python
def top_k(stream, k):
    heap = []
    for x in stream:
        if len(heap) < k:
            heapq.heappush(heap, x)
        else:
            heapq.heappushpop(heap, x)
            # 比堆顶小的会立刻被弹出 → 等于没进来
            # 比堆顶大的留下，旧堆顶被弹出
    return heap

print(top_k([3, 1, 5, 12, 2, 11, 7], 3))   # [7, 11, 12]
```

### 怎么记

- **`heappushpop`** = 名字怎么写就怎么做（push 再 pop）
- **`heapreplace`** = "替换" 语义，先扔旧的（pop）再放新的（push）

## 惰性删除模式（heap + dead 字典）

heap 不支持 O(log n) 任意删除（找元素就是 O(n)）。**用法**：把"删除"打个标记，等元素浮到堆顶时再 pop。

```python
from collections import defaultdict
import heapq

heap = []
dead = defaultdict(int)   # 值 -> 待删次数

# 加入
heapq.heappush(heap, x)

# 删除：只标记
dead[x] += 1

# 看堆顶/取最值前：清理死元素
while heap and dead[heap[0]] > 0:
    dead[heap[0]] -= 1
    heapq.heappop(heap)

# 现在 heap[0] 一定是活的
top = heap[0] if heap else None
```

### 关键：必须用 `while`，不能用 `if`

死元素可以**堆积**在堆顶下面：

```
        20         ← 活的，挡着下面
       / \
     15💀 18💀     ← 早就标记死了
     /
    10💀          ← 也死了
```

20 被 pop 之后，下面 15、18、10 全要清算。**一次操作可能清 0 个、1 个、也可能 10 个**，所以是 `while`。

### 复杂度（均摊分析）

- 单次清理 while 可能很贵（pop 多次）
- 但每个元素一生**只被 push 一次、pop 一次**
- 所有 while 加起来 ≤ N 次
- **总时间 O(N log N)**，渐进等价于"真删除"版本

|---|---|---|
| 218 | The Skyline Problem | heap + 惰性删除 |
| 480 | Sliding Window Median | 双堆 + 惰性删除 |
| 1606 | Find Servers That Handled Most Requests | heap + SortedList |
| 1851 | Minimum Interval to Include Each Query | heap + 惰性删除 |

## SortedList vs heap

| 操作 | heap | `sortedcontainers.SortedList` |
|---|---|---|
| 插入 | O(log n) | O(log n) 实际 O(√n) |
| peek 最值 | O(1) | O(log n) (`sl[0]` / `sl[-1]`) |
| pop 最值 | O(log n) | O(log n) |
| **删除任意元素** | ❌ O(n) | ✅ O(log n) |
| **查找任意元素** | ❌ O(n) | ✅ O(log n) |
| 按 rank 索引 `sl[k]` | ❌ | ✅ O(log n) |
| 范围查询 | ❌ | ✅ |
| 内存常数 | 小 | 较大 |

**口诀**：
- **heap = 极值专用**（只反复拿最值）
- **SortedList = 全序通用**（要任意删除/查找/按位置访问）

LeetCode Python 环境**自带 `sortedcontainers`**，可以直接用：

```python
from sortedcontainers import SortedList

sl = SortedList([3, 1, 4, 1, 5])
sl.add(2)            # O(log n) 插入
sl.remove(1)         # O(log n) 真删除
print(sl[0], sl[-1]) # 最小/最大 O(log n)
print(sl.bisect_left(3))  # 二分查找
```

## Stack 在 Python 里怎么用

Python **没有专门的 Stack 类**，直接用 `list` 就行。

```python
stack = []
stack.append(x)   # push，O(1)
stack.pop()       # pop 栈顶，O(1)
stack[-1]         # peek 栈顶
len(stack)        # 大小
not stack         # 判空
```

### 其他选项（一般不用）

- **`collections.deque`**：双端队列，两端 O(1)。**同时需要两端操作时**（如 BFS 用 `popleft`）才用。
- **`queue.LifoQueue`**：线程安全的栈，**多线程**场景才用，刷题别用，慢。

**刷题结论**：直接 `list` + `append` / `pop()`。

## Python 类型注解

Python 是动态语言，**类型注解运行时被忽略**，但能帮 IDE / 阅读者理解。

```python
stack: list[TreeNode] = []     # Python 3.9+，推荐
stack: List[TreeNode] = []     # 老写法，需 from typing import List
```

注意是 `list[TreeNode]`（方括号在 `list` 后面），**不是 `TreeNode[]`**（那是 TS 语法）。

### 常用注解

```python
from typing import Optional, List, Dict, Tuple

def f(x: int, name: str = "a") -> bool: ...
root: Optional[TreeNode]               # = TreeNode | None
nums: list[int]                        # 列表
mp: dict[str, int]                     # 字典
pair: tuple[int, str]                  # 元组
```

### 刷题实践

- **函数签名**：写注解（保持和 LeetCode 模板一致）。
- **局部变量**：基本不写，靠命名表意。`stack = []`、`cur = root` 就够了。

## Truthy / Falsy：Python vs JS 对比

判断真假时，**哪些值算 falsy** 在两门语言里**不一样**，这是最容易踩坑的地方。

### Python falsy 值
- `None`
- `0`、`0.0`
- `""`（空字符串）
- `[]`、`{}`、`set()`（**空容器**）
- `False`

### JS falsy 值
- `null`、`undefined`
- `0`、`NaN`
- `""`
- `false`

**重点**：JS 里**空数组 `[]` 和空对象 `{}` 都是 truthy**，Python 里却是 falsy。

### ⚠️ 常见误解澄清

> "JS 里 `[]` 是 truthy，是不是因为它是引用对象？Python 里是不是不是引用？"

**两边都是引用对象**，list / dict / array / object 在 Python 和 JS 里**都**有内存地址、都是按引用传递：

```python
# Python
a = [1, 2, 3]
b = a
b.append(4)
print(a)        # [1, 2, 3, 4]  ← 同一个对象
print(a is b)   # True
```

```js
// JS：行为完全一样
let a = [1, 2, 3];
let b = a;
b.push(4);
console.log(a); // [1, 2, 3, 4]
```

**差异只是"怎么定义真假"，跟引用语义无关**：

- **Python 设计哲学**："是否有内容"才是真假。`list` / `dict` / `set` 都实现了 `__len__` 或 `__bool__`，长度为 0 就是 falsy。
- **JS 设计哲学**：只有 6 种特定值是 falsy（`false / 0 / "" / null / undefined / NaN`），**任何对象**（不管空不空）都是 truthy。JS 不会调用对象方法去判真假。

```python
# Python 机制
class MyList:
    def __len__(self): return 0
bool(MyList())   # False ← __len__ 被调用了
```

```js
// JS 机制
Boolean([])                   // true
Boolean({})                   // true
Boolean(new Boolean(false))   // true ❗ 连这个都 true
```

| | Python | JS |
|---|---|---|
| list / dict 是引用对象？ | ✅ 是 | ✅ 是 |
| 空容器 truthy/falsy？ | **falsy**（看内容） | **truthy**（只要是对象） |

### 对照表

| 表达式 | Python | JS |
|---|---|---|
| 空 list / array | `[]` → **falsy** ✅ | `[]` → **truthy** ❗ |
| 空 dict / object | `{}` → **falsy** ✅ | `{}` → **truthy** ❗ |
| 空字符串 `""` | falsy | falsy |
| `0` | falsy | falsy |
| `None` / `null` | falsy | falsy |

### 实战：判空数组

```python
# Python：随便写
if not stack: ...
while cur or stack: ...
```

```js
// JS：不能直接 if (stack)！
if (stack.length === 0) { ... }
if (!stack.length) { ... }
while (cur !== null || stack.length > 0) { ... }
```

### 写法对比（94 题迭代中序）

```python
# Python 简洁版
while cur or stack:
    while cur:
        stack.append(cur)
        cur = cur.left
    cur = stack.pop()
    res.append(cur.val)
    cur = cur.right
```

```typescript
// TS：stack 必须用 .length
while (cur !== null || stack.length > 0) {
    while (cur !== null) {
        stack.push(cur);
        cur = cur.left;
    }
    cur = stack.pop()!;
    res.push(cur.val);
    cur = cur.right;
}
```

### 关于 `if x` vs `if x is not None`

- 对引用对象（TreeNode、ListNode 等）写 `if cur` 完全 OK。
- 对**可能是 0 / "" / [] 的变量**，要严格用 `is not None`，避免把合法值误判为空：

```python
count = 0
if count: ...         # ❌ 不进入，0 是 falsy
if count is not None: # ✅ 进入
```

JS 里同理：`0`、`""` 是 falsy，对数字/字符串变量判存在用 `!== null` / `!== undefined` 更稳。
