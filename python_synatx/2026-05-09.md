## Python: 初始化 set / list / dict

```python
# set
s = set()           # 空 set
s = {1, 2, 3}       # 有元素
s = set([1, 2, 3])  # 从 list 转

# list (array)
arr = []
arr = [1, 2, 3]

# dict
d = {}              # 空 dict（{} 是 dict 不是 set）
d = {"a": 1, "b": 2}
d = dict()
```

⚠️ `{}` 是空 dict，空 set 必须用 `set()`。

## Python: 什么数据可以转成 set

任何**可迭代**（iterable）且元素**可哈希**（hashable）的数据：

```python
set([1, 2, 3])        # list
set((1, 2, 3))        # tuple
set("hello")          # string → {'h','e','l','o'}
set({"a":1, "b":2})   # dict → {'a','b'}（只取 key）
set(range(5))         # range → {0,1,2,3,4}
```

**不可哈希的元素**不能放进 set（list、dict、set 本身）：

```python
set([[1,2], [3,4]])   # ❌ TypeError: list 不可哈希
set([(1,2), (3,4)])   # ✅ tuple 可以
```

记忆：不可变（immutable）的基本都可哈希。

---

## JS: 什么数据可以转成 Set

`new Set()` 接受任何**可迭代**（iterable）的数据：

```javascript
new Set([1, 2, 3])           // Array
new Set("hello")             // String → {'h','e','l','o'}
new Set(new Map([['a',1]]))  // Map → 存的是 [key, value] 对
new Set(new Set([1,2]))      // Set
```

和 Python 不同，JS 的 Set **任何值都能存**（包括对象、数组），因为它用引用比较，不需要"可哈希"：

```javascript
const s = new Set();
s.add({a: 1});      // ✅ 对象可以
s.add([1, 2]);      // ✅ 数组可以
s.add(NaN);         // ✅ NaN 也可以（且只存一份）
```

⚠️ 对象比较是按**引用**：

```javascript
const s = new Set();
s.add({a: 1});
s.add({a: 1});  // 又加了一个，size = 2（不同引用）
```

## 什么是 iterable

**iterable**（可迭代）= 可以被 `for ... in` / `for ... of` 遍历的对象。

技术上：实现了 `__iter__`（Python）/ `[Symbol.iterator]`（JS）的对象。

```python
for x in [1,2,3]:    # list 可迭代
for x in "abc":      # string 可迭代
for x in {1,2,3}:    # set 可迭代
for x in 123:        # ❌ int 不可迭代
```

## 为什么 string 是 iterable

string 本质是**字符序列**，可以一个字符一个字符地拿出来：

```python
for c in "hello":
    print(c)
# h e l l o
```

所以 `set("hello")` 把每个字符依次加进 set，得到 `{'h','e','l','o'}`。

简单判断：能 `for x in 它` 就是 iterable。常见的：list、tuple、string、set、dict、range、generator。

## Python vs JS 的 iterable 区别

**共同点**：list/array、string、set、map/dict 都是 iterable，都能 for 遍历。

**1. 语法不同**

```python
for x in arr:        # Python
```
```javascript
for (const x of arr)   // JS：用 of 不是 in
for (const x in obj)   // JS：in 是遍历 key（属性名）
```

⚠️ JS 里 `for...in` 和 `for...of` 是**两个不同的东西**，Python 没这区别。

**2. 普通对象**

```javascript
const obj = {a: 1, b: 2};
for (const x of obj)   // ❌ 报错，普通对象不是 iterable
for (const k in obj)   // ✅ 遍历 key
```

Python 的 dict 直接可迭代（默认遍历 key）。

**3. 实现机制**

- Python：实现 `__iter__`
- JS：实现 `[Symbol.iterator]`

## JS 遍历 string 的几种写法

```javascript
const s = "hello";

// 1. for...of（最常用，拿到字符）
for (const c of s) console.log(c);

// 2. 普通 for（拿到索引）
for (let i = 0; i < s.length; i++) {
    console.log(s[i]);
}

// 3. forEach（先转数组）
[...s].forEach(c => console.log(c));

// 4. for...in（拿到索引，不推荐用于 string/array）
for (const i in s) console.log(s[i]);
```

最推荐 `for...of`，简洁直观。

## iterable vs indexable（Python 和 JS 一样）

**iterable ≠ 可以用 index**。

### 可以用 index（支持 `arr[i]`）— 有序序列

| 类型 | Python | JS |
|------|--------|-----|
| list / array | ✅ | ✅ |
| tuple | ✅ | — |
| string | ✅ | ✅ |

### 不能用 index（虽然 iterable）

| 类型 | 原因 |
|------|------|
| set | 无序，没有"第几个" |
| dict / object | 用 key 访问，不是 index |
| generator | 一次性流，只能往前遍历 |

```python
s = {1, 2, 3}
s[0]            # ❌ TypeError: 'set' object is not subscriptable
for x in s:     # ✅ 可以遍历
    print(x)
```

```javascript
const s = new Set([1,2,3]);
s[0]            // undefined（不报错，但拿不到）
for (const x of s) console.log(x);  // ✅
```

### Python vs JS 区别

- **Python**：set/dict 用 `s[0]` 直接 `TypeError`
- **JS**：Set/Map 用 `s[0]` 返回 `undefined`（不报错但拿不到）

逻辑上一致：set/dict 都不支持 index 访问。

### 记忆

- **iterable** = 能遍历（一个一个拿）
- **indexable / subscriptable** = 能随机访问（直接拿第 i 个）

后者是前者的子集：能 index 的一般都 iterable，但 iterable 不一定能 index。

## Python dict 操作 vs JS Map

| 操作 | JS Map | Python dict |
|------|--------|-------------|
| 创建空 | `new Map()` | `{}` |
| 设置 | `m.set(k, v)` | `m[k] = v` |
| 取值 | `m.get(k)` | `m[k]` 或 `m.get(k)` |
| 判断存在 | `m.has(k)` | `k in m` |
| 删除 | `m.delete(k)` | `del m[k]` |
| 大小 | `m.size` | `len(m)` |

⚠️ Python dict 没有 `.set()` 方法，要用 `d[k] = v`。

⚠️ `m[k]` 取不存在的 key 会**抛 KeyError**；`m.get(k)` 返回 `None`（更安全）；`m.get(k, default)` 可以指定默认值。

## Python `.get(key, default)` 的巧妙用法

用**当前值**作为默认值，让"没见过"的 key 自动判等通过：

```python
# LC 290 紧凑写法
for c, w in zip(pattern, words):
    if c2w.get(c, w) != w or w2c.get(w, c) != c:
        return False
    c2w[c] = w
    w2c[w] = c
```

### 拆解 `c2w.get(c, w) != w`

- **`c` 已见过** → 返回之前存的值，跟 `w` 比较：不等就冲突 → false
- **`c` 没见过** → 返回默认值 `w`，`w != w` 是 false → 不触发 return，进入下一步建映射

### 等价的更直白写法

```python
if (c in c2w and c2w[c] != w) or (w in w2c and w2c[w] != c):
    return False
```

### 关键直觉

> 默认值设成"和当前值相等的东西"，新 key 永远过得了等值检查。

省掉了 `in` 检查，是 Python idiomatic 的小技巧。

## Python `zip()` —— 并行遍历多个序列

`zip()` 把多个 iterable **按位置配对**，每次产出一个元组。

```python
pattern = "abba"
words = ["dog", "cat", "cat", "dog"]

for c, w in zip(pattern, words):
    print(c, w)
# a dog
# b cat
# b cat
# a dog
```

### 为什么好用：避免下标循环

```python
# 啰嗦写法
for i in range(len(pattern)):
    c = pattern[i]
    w = words[i]

# zip 写法（更 Pythonic）
for c, w in zip(pattern, words):
    ...
```

### 几个细节

**1. 长度不一致 → 取最短**
```python
list(zip([1, 2, 3], ['a', 'b']))
# [(1, 'a'), (2, 'b')]   ← 第三个 3 被丢掉
```

**2. 可以传 3 个或更多**
```python
for a, b, c in zip(list1, list2, list3):
    ...
```

**3. 配合 enumerate**
```python
for i, (c, w) in enumerate(zip(pattern, words)):
    print(i, c, w)
```

**4. 严格模式（Python 3.10+）**
```python
zip(a, b, strict=True)  # 长度不等会抛 ValueError
```

### vs JS

JS 没有内置 `zip`，得手写：
```javascript
arr1.map((x, i) => [x, arr2[i]])
```
Python 的 `zip` 处理"两个等长序列并行遍历"非常顺手，是日常神器。

### zip 返回的是什么？—— 惰性迭代器（不是 list）

```python
z = zip([1, 2, 3], ['a', 'b', 'c'])
print(z)         # <zip object at 0x...>
print(type(z))   # <class 'zip'>
```

要拿到 list of tuples，得显式转：
```python
list(zip([1,2,3], ['a','b','c']))
# [(1, 'a'), (2, 'b'), (3, 'c')]
```

⚠️ **迭代器只能用一次**：
```python
z = zip([1,2], ['a','b'])
list(z)   # [(1, 'a'), (2, 'b')]
list(z)   # []  ← 已经耗尽
```

## 惰性迭代器（Lazy Iterator）—— Python 核心概念

### 一句话

> **惰性 (lazy) = 用到才算，不用不算。**

跟 list "一次性把所有元素都算出来塞进内存" 完全相反。

### 直观对比

```python
# list：立刻生成所有元素
nums = [x * 2 for x in range(1_000_000)]
# 立刻占好几 MB 内存

# 惰性迭代器：什么也没算，只是"承诺"以后能给你
gen = (x * 2 for x in range(1_000_000))
# 几乎不占内存，一个数都还没算
```

只有当你"问它要"的时候才算：

```python
next(gen)   # 0      ← 现在才算第一个
next(gen)   # 2      ← 现在才算第二个
```

### 比喻：自助餐 vs 点菜

| | list | 惰性迭代器 |
|---|---|---|
| 比喻 | **自助餐** | **点菜** |
| 时机 | 一次性摆上桌 | 你点一道做一道 |
| 内存 | 占满整张桌 | 桌上只有当前那盘 |
| 重复 | 还能再夹 | 吃完就没了 |

### zip 的"惰性"具体怎么发生

```python
z = zip([1,2,3], ['a','b','c'])
# 此时：什么都没算

for pair in z:
    print(pair)
# 第 1 轮：才计算 (1, 'a')
# 第 2 轮：才计算 (2, 'b')
# 第 3 轮：才计算 (3, 'c')
```

### 三个关键性质

**1. 省内存** —— 同时只存当前那一个元素
```python
for x, y in zip(huge_a, huge_b):
    ...   # 内存里同时只有一个 (x, y) 对
```

**2. 只能用一次（耗尽后就空了）** —— 它没存东西，只是个"指针"

**3. 可以提前停止** —— 只算你需要的部分
```python
for x, y in zip(huge_a, huge_b):
    if x > 100:
        break   # 后面的元素根本不会被计算
```

### Python 里其他惰性的东西

| 写法 | 是否惰性 |
|------|---------|
| `range(n)` | ✅ 惰性 |
| `map(f, lst)` | ✅ 惰性 |
| `filter(f, lst)` | ✅ 惰性 |
| `zip(a, b)` | ✅ 惰性 |
| `(x for x in lst)` 生成器 | ✅ 惰性 |
| `for line in open(file)` | ✅ 一行一行读 |
| `[x for x in lst]` 列表推导 | ❌ 立即求值 |

### 强制"立即求值" —— 套 `list()`

```python
list(zip(a, b))
list(map(str, [1,2,3]))
list(range(10))
```

### 一句话总结

> **list = 先全做好放着；迭代器 = 现做现给。**
>
> 好处：省内存、能 break、能处理无限序列
> 代价：只能用一次、不知道总长度

## Python: `s.split()` vs `s.split(' ')`

```python
words = s.split(' ')   # 严格按单个空格切，连续空格 → 空字符串
words = s.split()      # 按任意空白（空格/tab/换行），自动忽略多余空格
```

不传参数的 `s.split()` 更常用、更安全。

## `defaultdict(int)` —— 自动默认值的字典

来自标准库 `collections`。访问不存在的 key 时，**自动用工厂函数生成默认值**，不抛 `KeyError`。

```python
from collections import defaultdict

d = defaultdict(int)
print(d['x'])      # 0  （自动创建 d['x']=0）
d['y'] += 1        # 等价于 d['y'] = 0 + 1
```

对比普通 dict：
```python
d = {}
d['x'] += 1        # ❌ KeyError
d['x'] = d.get('x', 0) + 1   # 普通 dict 要这样写
```

### 常见工厂函数

| 写法 | 默认值 | 用途 |
|------|--------|------|
| `defaultdict(int)` | `0` | 计数 / 累加 |
| `defaultdict(list)` | `[]` | 分组（值是列表）|
| `defaultdict(set)` | `set()` | 分组去重 |
| `defaultdict(str)` | `''` | 字符串拼接 |

### 用在前缀和 + 哈希（LeetCode 560）

```python
from collections import defaultdict

def subarraySum(nums, k):
    count, prefix_sum = 0, 0
    freq = defaultdict(int)
    freq[0] = 1
    for num in nums:
        prefix_sum += num
        count += freq[prefix_sum - k]   # 不存在自动 0，安全
        freq[prefix_sum] += 1            # 不用 .get()
    return count
```

### ⚠️ 一个坑：读取也会创建 key

```python
freq = defaultdict(int)
if freq[x] > 0:    # 这一行会无声地创建 freq[x] = 0！
    ...
```

要"只检查存在性"用 `if x in freq:`，不要用 `freq[x]`。

### 内存开销

`defaultdict` 是 `dict` 的子类，底层就是 dict，**几乎没有额外开销**（只多存一个工厂函数指针）。
真正的内存风险来自上面那个坑——误用读取会塞进一堆 `key:0`。

### 面试推荐度

✅ **强烈推荐**——标准库、Pythonic、代码更短。计数场景无脑用 `defaultdict(int)`。
（计数也可以用 `Counter`，但 `defaultdict` 更通用。）
