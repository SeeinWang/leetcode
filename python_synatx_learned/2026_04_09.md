# Python 语法笔记 - 2026/04/09

## 1. Array 中两个位置的值交换

```python
nums[i], nums[j] = nums[j], nums[i]
```

不需要临时变量，Python 原生支持同时赋值。

## 2. if-elif-else

```python
if condition1:
    # ...
elif condition2:
    # ...
elif condition3:
    # ...
else:
    # ...
```

注意：用 `elif` 而不是 `else if`。如果用多个 `if`，每个条件都会被检查；用 `elif` 则命中一个后跳过其余分支。

## 3. List Comprehension（列表推导式）

```python
# 对每个元素做平方
[num * num for num in nums]

# 反向遍历并平方（reversed 返回迭代器，O(1) 创建，不会真的反转数组）
[num * num for num in reversed(nums)]
```

等价于 JS 的 `nums.map(num => num * num)`。Python 中更推荐用 list comprehension 而不是 `map()`。

## 4. Python list 不能对超出范围的索引赋值

```python
# Python - 报错 IndexError
arr = []
arr[5] = 10  # ❌

# 需要预分配大小
arr = [0] * 6
arr[5] = 10  # ✅
```

JS 中数组可以直接对任意索引赋值，数组会自动扩展；Python 的 list 不行，索引必须在已有范围内。

## 5. Set 的基本操作

```python
# 创建一个空的 set（不需要 new 关键字）
num_set = set()

# 检查元素是否在 set 里
if num in num_set:

# 往 set 里添加元素
num_set.add(num)
```

注意：Python 命名用 snake_case（如 `num_set`），不用 camelCase。

## 6. Python 没有 `new` 关键字

```python
num_set = set()
num_list = list()
num_dict = dict()
obj = MyClass()
```

所有类型都直接调用，不需要 `new`。

## 7. 字面量语法 vs 构造函数

```python
a = []       # 等价于 list()，更常用
d = {}       # 等价于 dict()，更常用
s = {1, 2}   # 有初始值的 set
```

注意：`{}` 创建的是**字典**不是 set，空 set 只能用 `set()`。

## 8. Python vs JS 创建对象的对比

| 用途 | Python | JS |
|------|--------|----|
| Set | `set()` | `new Set()` |
| Dict / Map | `dict()` 或 `{}` | `new Map()` 或 `{}` |
| List / Array | `list()` 或 `[]` | `new Array()` 或 `[]` |
| 自定义类 | `MyClass()` | `new MyClass()` |

Python 全部不需要 `new`，JS 只有数组和对象有字面量语法可以省掉 `new`。

## 9. dict 计数：get(key, default)

统计字符频率时，用 `dict.get(key, default)` 代替 if/else：

```python
# 不够 Pythonic
for s in magazine:
    if s in maga_map:
        maga_map[s] = maga_map[s] + 1
    else:
        maga_map[s] = 1

# Pythonic：get 有值就返回值，没有就返回默认值 0
for s in magazine:
    maga_map[s] = maga_map.get(s, 0) + 1
```

对应 JS 的写法：
```js
// JS
for (const c of magazine) {
    map.set(c, (map.get(c) || 0) + 1);
}
```

Python 用 `dict.get(key, 0)`，JS 用 `map.get(key) || 0`，效果一样。

## 10. 删除元素：del / remove / discard

### Dict（对应 JS Map）

```python
# Python
d = {"a": 1, "b": 2}
del d["a"]        # 删除 key，不存在会报 KeyError
# 结果：{"b": 2}
```

```js
// JS Map
const m = new Map([["a", 1], ["b", 2]]);
m.delete("a");    // 删除 key，不存在不报错，返回 false

// JS Object
const obj = { a: 1, b: 2 };
delete obj.a;     // 结果：{ b: 2 }
delete obj["a"];  // 同上，方括号写法，不存在也不报错
```

### List（对应 JS Array）

```python
# Python
arr = [1, 2, 3]
del arr[1]        # 按索引删，不存在会报 IndexError → [1, 3]
arr.remove(2)     # 按值删，删第一个匹配的，不存在会报 ValueError
```

```js
// JS 没有直接按值删除的方法
arr.splice(1, 1)          // 按索引删
arr = arr.filter(x => x !== 2)  // 按值删（返回新数组）
```

### Set

```python
# Python
s = {1, 2, 3}
s.remove(2)    # 值不存在会报 KeyError
s.discard(2)   # 值不存在不报错，做题推荐用这个
```

```js
// JS
const s = new Set([1, 2, 3]);
s.delete(2);   // 值不存在不报错，返回 false（类似 Python 的 discard）
```

### 对比总结

| | Python dict | Python list | Python set |
|---|---|---|---|
| 删除 | `del d[key]` | `del arr[i]` 或 `arr.remove(val)` | `s.remove(val)` 或 `s.discard(val)` |
| 不存在时 | `KeyError` | `IndexError` / `ValueError` | `KeyError` / 静默 |

JS 的 `Map.delete()` 和 `Set.delete()` 不存在时都不报错；Python 要用 `discard()` 才能做到同样效果。

## 11. 运算符优先级：算术 vs 逻辑

算术运算符（`* / + -`）的优先级比逻辑运算符（`|| &&` / `or and`）**高**。

```js
// JS 例子
x || 0 + 1     // ❌ 实际是 x || (0+1)，先算加法
(x || 0) + 1   // ✅ 先算 ||，再加 1
```

```python
# Python 对比
x or 0 + 1     # ❌ 实际是 x or (0+1)，同样先算加法
(x or 0) + 1   # ✅ 先算 or，再加 1
```

Python 的 `and` / `or` 和 JS 的 `&&` / `||` 优先级规则一样：算术运算符先算。需要改变顺序就加括号。
